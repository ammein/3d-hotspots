## Getting Started

### Installation
1. First install the package:

<CodeBlock copyToClipboard language='bash'>
{`# Install Package
yarn Install`}
</CodeBlock>

2. Then, fetch Lygia package:

<CodeBlock copyToClipboard language='bash'>
{`# Fetch Lygia Package
yarn lygia`}
</CodeBlock>
> When prompt to prune the file, type to choose `glsl`

3. Then, fetch basis package:

<CodeBlock copyToClipboard language='bash'>
{`# Fetch ThreeJS Basis Package
yarn basis
`}
</CodeBlock>

3. Then finally, fetch draco package:

<CodeBlock copyToClipboard language='bash'>
{`# Fetch Draco Package
yarn draco
`}
</CodeBlock>

### Start App
<CodeBlock copyToClipboard language='bash'>
{`# Finally run dev server
yarn dev
`}
</CodeBlock>

### Deploy
First you must create a file in root folder of the project called `.env.production`:
<CodeBlock language="diff" showLineNumbers style={prism}>
{`.
    ├── ./.env.development
    ├── ./.env.preview
+├── ./.env.production
    ├── ./.gitignore
    ├── ./.gitlab-ci.yml
    ├── ./.prettierignore
    ├── ./.prettierrc
    ├── ./.storybook
    ├── ./.vscode
    ├── ./3D Hotspot.code-profile
    ├── ./readme-assets
    ├── ./eslint.config.js
    ├── ./index.html
    ├── ./jsconfig.json
    ├── ./package.json

    ├── ./src
    ├── ./vite.config.js
    └── ./yarn.lock

9 directories, 24 files`}
</CodeBlock>

In your `.env.production` file:
<CodeBlock copyToClipboard showLineNumbers wrapLongLines language='bash'>
{`# Model Name
VITE_MODEL_NAME=<your-model-name>
# Model URL
VITE_BASE_URL=https://production-domain.com/$VITE_MODEL_NAME
# SEO itemtype='http://tracking.domain.com/'
# More info: https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Global_attributes/itemscope
VITE_TRACKING_URL=http://tracking.domain.com/
# DRACO path in public folder after run 'yarn draco' & to locate public folder to Draco Folder
VITE_LOCAL_DRACO_PATH=three/draco/javascript/
# Basis path in public folder after run 'yarn basis' & to locate public folder to Basis Folder
VITE_LOCAL_KTX_PATH=examples/jsm/libs/basis/
# To disable sourcemap in production
VITE_SOURCEMAP=false
# IMPORTANT for Interactive Asset export.
# (This will compile all files in one single Javascript module file so that it can be played in 3DS Webpage)
VITE_DEPLOY=true`}
</CodeBlock>

> You also need to configure the same SEO value for `tracking` in [metadata configuration](?path=/docs/documentation-user-metadata--docs#trackingcategory)

Then you will encounter to define ratio for Desktop responsive:
<CodeBlock language='bash'>
{`Your bundle desktop ratio is "16 / 9". Confirm? (y/n)
>`}
</CodeBlock>
> If you wish to use default ratio for desktop, simply answer by typing `y` and press `Enter`

If you answer `no`, it will prompt an output as below and you will have to type your own ratio in this format `width / height` as per documented in [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/aspect-ratio):
<CodeBlock language='bash'>
{`Write your custom ratio in this format. Tips:
- Replace '<width>' & '<height>' to your desired value
- For more info: https://developer.mozilla.org/en-US/docs/Web/CSS/aspect-ratio

Ex: <width> / <height>

>`}
</CodeBlock>

> Then it will prompt the same output for mobile responsive. You can adjust the value as per needed. 

#### Final Output
Then, the Vite will compiled and build folder to your destination folder that were set in `VITE_MODEL_NAME` as folder name. Below is the example output console:
<CodeBlock language='bash'>
{`----------------------------------------------
Model Name: <model-name>
Model URL: https://domain.com/<model-name>/index-<hashed>.js
Bundle Name: <bundle-name>
----------------------------------------------`}
</CodeBlock>

### Handover
<Paragraph weight='regular' className='list-decimal !text-[16px]' >1. ZIP/Compress the file and handoff to your server admin to put the folder in server.</Paragraph>
